### CIFAR-10 画像分類モデルの検証（PyTorch）

PyTorchを用いて、CIFAR-10データセットを分類するモデルを構築・検証したリポジトリです。

### 本リポジトリの目的
* 東京大学 松尾研究室が主催する「DL基礎講座」で学んだ基礎知識を、実際のフレームワークを用いてゼロから実装すること
* ゼロから作るDeep Learning①で学んだ画像認識における様々な手法を、自分の手でコードに落とし込むことで、手法による変化を実践的に分析すること。

### 実装の特徴と試行錯誤のプロセス

初期構造として、LeNetベースのモデル（畳み込み層2層、全結合層3層）からスタートしました。
その後、書籍や講義で学んだことを活かすため、自力で以下の拡張機能を導入して検証しました。

＜最適化手法の選定＞
* 初期段階から**Adam**を採用し、効率的な学習を行いました。
＜重みの初期化＞
* PyTorchのデフォルトの初期化に頼らず、ReLU関数と相性の良い**Heの初期値**を導入しました。
＜バッチ正規化＞
* 各層の勅語にBatch Normalizationをハサミ、ミニバッチごとのデータの偏りを抑える実験を行いました。
＜ネットワークの拡張＞
* モデルの表現力を大幅に高めるため、畳み込み層のフィルター数を6チャンネルから32チャンネルに拡張しました。
＜エポック数の調整＞
* 初期検証の1エポックから10エポック、さらに学習回数を増やしてテスト精度を観察しました。
＜過学習対策＞
* 過学習を疑い、全結合層にDropoutを導入して汎化性能の向上を試みました。

### 検証結果

**初期状態正解率**：62.71%
**最高正解率**：63.24%
**機能追加・実験を重ねた後の正解率**：58.70 ~ 63.24%

### 考察
実験結果として、初期のLeNetベースにバッチ正規化を導入した段階がもっとも高い正解率を記録しました。
その後、Droppoutやチャンネル数を重ねることで、かえって制度が50%台まで下がってしまうことが確認できました。

一連の検証が終わった段階で改めて振り返り、書籍の勉強中に、「Dropoutとバッチ正規化は相性が悪いとさらっと調べたことがあった」ことを思い出しました。当時は座学として聞き流してしまい、実際の実装でそれを活かすことができませんでした。

今回の実験を通してすべての手法を闇雲に重ねるのではなく、各技術の特性や相性を考慮し、時には「手法を引き算する」ことの重要性を深く学ぶことができました。
